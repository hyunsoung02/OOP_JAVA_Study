## 변수 
### 변수(Variable)
변수: 하나의 값을 저장할 수 있는 **기억공간**

* 기본형 (Primitive type)

   * 8개(boolean, char, byte, short, int, lonh, float, double)
   * 실제 값을 저장
 
* 참조형 (Referemce type)
   * 기본형을 제외한 나머지 (String, System 등)
   * 객체의 주소를 저장 (4 bype, 0x00000000~ 0xffffffff)

논라형 - true와 false중 하나를 값으로 갖으며, 조건식과 논리적 계산에 사용된다.

문자형 - 문자를 저장하는데 사용되먀, 변수 당 하나의 문자만을 저장할 수 있다

정수형 - 정수 값을 저장하는데 사용된다. 주로 사용하는 것은 int와 long이며, byte는 이진데이터를 다루는데 사용되며, short은 c언어와 호환을 위해 추가되었다.

실수형 - 실수 값을 저장하는데 사용된다. float와 double이 있다.

| 실수형 | - | - | float | double |

### 형변환

 * 값의 타입을 다른 타입으로 변환하는 것이다.
 * boolean을 제외한 7개의 기본형은 서로 형변환이 가능하다.

| 변환 | 수식 | 결과 |
| :--: | :--: | :--: |
|int -> char | (char) 65 | 'A' |
|char -> int | (int) 'A' | 65 |
|float -> int | (int) 1.6f | 1 |
|int -> float | (float) 10 | 10.0f |

1. byte -> int

byte b =10

int i = (int) b; // 생략가능


2. int -> byte

int i2 =300

byte b2 = (byte)i2; // 생략불가

 * 크기가 작은 타입에서 큰 타입으로 변환하는 경우에는 형변환을 생략가능
 * 큰 타입에서 작은 타입으로 변환한 경우에는 값손실이 발생할 수 있지만 작은 타입에서 큰 타입으로 변환하는 경우에는 값손실이 발생하지 않기 때문에 형변환을 자동적으로 수행하도록 허용하는 것입니다.

### 삼항연산자의 특징 - ? :

      (조건식) ? 식1 : 식2

##  조건문과 반복문
### 조건문의 특징
* 조건문은 조건식과 실행될 하나의 문장 또는 블럭 {}으로 구성
* JAVA에서 조건문은 if문과 switch문 두 가지 뿐이다.
* if문이 주로 사용되며, 경우의 수가 많은 경우 switch문을 사용할 것을 고려한다.
* 모든 switch문은 if문으로 변경이 가능하지만, if문은 switch문으로 변경할 수 없는 경우가 많다.
* if문
   * if문은 if, if-else, if-else if의 세가지 형태가 있다.
   * 조건식의 결과는 반드시 true 또는 false이어야 한다.
* 중첩 if문
   * if문 안에 또 다른 if문을 중첩해서 넣을 수 있다.
   * if문의 중첩횟수에는 거의 제한이 없다.
* switch문
    * if문의 조건식과 달리, 조건식의 계산결과가 int범위 이하의 정수만 가능
    * 조건식의 계산결과와 일치하는 case문으로 이동 후 break문을 만날 때까지 문장들을 수행한다.(break문이 없으면 switch문의 끝까지 진행)
    * 일치하는 case문의 값이 없는 경우 default문으로 이동한다. ( default문 생략가능 )
    * case문의 값으로 변수를 사용할 수  없다. ( 리터럴, 상수만 가능)
* 중첩 switch문
    * switch문 안에 또 다른 switch문을 중첩해서 넣을 수 있다.
    * switch문의 중첩횟수에는 거의 제한이 없다.

if문과 switch문의 비교

if문이 주로 사용되며, 경우의 수가 많은 경우 switch문을 사용할 것을 고려한다. 

모든 switch문은 if문으로 변경이 가능하지만, if문은 switch문으로 변경할 수 없는 경우가 많다. 

if문 보다 switch문이 더 간결하고 효율적이다. 

### 반복문의 특징
* 문장 또는 문장들을 반복해서 수행할 때 사용
* 조건식과 수행할 블록 {} 또는 문장으로 구성
* 반복회수가 중요한 경우에 for문을 그 외에는 while문을 사용하다.
* for문과 while문은 서로 변경가능하다.
* do-while문은 while문의 변형으로 블록 {}이 최소한 한 번은 수행될 것을 보장한다.
* for문
   * 초기화, 조건식, 증감식 그리고 수행할 블록 {} 또는 문장으로 구성
* 중첩for문
   * for문 안에 또 다른 for문을 포함시킬 수 있다.
   * for문의 중첩횟수에는 거의 제한이 없다.
* while문
   * 조건식과 수행할 블록 {} 또는 문장으로 구성
  
  while (조건식) {

  // 조건식의 연산결과가 true일 때 수행될 문장들을 적는다.

  }

  
  * 중첩while문
   * while문 안에 또 다른 while문을 포함시킬 수 있다.
   * while문의 중첩횟수에는 거의 제한이 없다.
 
  **break문**
  자신이 포함된 하나의 반복문 또는 switch문을 빠져 나온다.

  주로 if문과 함께 사용해서 특정 조건을 만족하면 반복문을 벗어나게 한다.

  **continue문**
  자신이 포함된 반복문의 끝으로 이동한다.( 다음 반복으로 넘어간다.)

  continue문 이후의 문장들은 수행되지 않는다.

  반복문 앞에 이름을 붙이고, 그이름을 break, continue와 같이 사용함으로써 둘 이상의 반복문을 벗어나거나 반복을 건너뛰는 것이 가능한다.

### 배열(Array)

같은 타입의 여러 변수를 하나의 묶음으로 다루는 자료구조이다. 변수가 많아질 때 하나하나 선언하는 번거로움을 해결해 준다.

* 배열의 선언과 생성
    * 선언: int[] score; (컴퓨터에게 배열을 다룰 참조변수 자리를 만들라고 명하는 것)
    * 생성: score = new int[5]; (실제 메모리에 값을 저장할 5개의 공간을 확보하는 것)

* 인덱스(Index)의 범위

배열의 각 방은 0부터 시작한다.

인덱스의 범위는 0부터 배열길이 - 1까지다.
     * 주의할 점: 만약 범위를 벗어난 인덱스를 사용하면 ArrayIndexOutOfBoundsException 에러가 발생하므로 주의해야 한다.

* 배열의 길이 (배열이름.length)

   *  자바에서는 score.length를 통해 배열의 길이를 가져올 수 있다.
   * 배열은 한 번 생성되면 메모리 공간의 크기를 절대 변경할 수 없다(상수 형태). 따라서 크기를 늘리고 싶다면 더 큰 배열을 새로 만들고 기존 내용을 복사해야 한다.

### 배열의 초기화와 출력

* 배열을 생성하면 기본적으로 해당 타입의 기본값(int는 0, 참조형은 null)으로 자동 초기화된다.

* 생성물과 동시에 초기화하기

int[] score = new int[]{ 100, 90, 80, 70, 60 };

int[] score = { 100, 90, 80, 70, 60 }; // 'new int[]'는 생략이 가능하다.

배열의 모든 값 출력하기

* 보통 for문을 사용해 한 방씩 출력한다.

* 더 간단하게 배열 내부 값을 통째로 확인하고 싶을 때는 자바가 제공하는 함수를 사용하면 된다:
      * System.out.println(Arrays.toString(score)); -> [100, 90, 80, 70, 60] 형태로 한눈에 출력됨.

### 다차원 배열 (2차원 배열)

 테이블 형태의 데이터(행과 열)를 다루기 위해 배열 속에 배열을 또 넣는 개념이다.

* 선언과 생성
   *  int[][] score = new int[4][3]; -> 4행 3열짜리 총 12개의 int 공간을 만든다.
* 메모리 구조의 핵심
   * 자바에서 2차원 배열은 사실 '배열의 배열' 구조다.
   * score라는 큰 배열의 각 방이 다시 소형 배열의 주소값(참조변수)을 가리키는 형태로 이루어져 있다.
   * 그렇기 때문에 행마다 열의 길이가 다른 '가변 배열'도 생성할 수 있다.
* 2차원 배열의 접근
   * 2차원 배열의 모든 요소를 다루려면 반드시 중첩 for문(2중 for문)을 사용해야 한다. (바깥쪽 for문은 '행', 안쪽 for문은 '열'을 가리킴)

### String 배열과 String 클래스

자바에서 가장 많이 쓰는 문자열 배열의 특징이다.

* 참조형 배열
    * String[] name = new String[3]; 처럼 참조형 변수를 담는 배열을 만들면, 각 방에는 실제 문자열 데이터가 아니라 문자열이 저장된 메모리의 주소(참조값)가 들어간다.
    * String 클래스는 char 배열의 확장판이다
    *  자바에서 String(문자열)은 단순한 문자의 배열(char[])이지만, 여기에 문자를 다루기 편리한 여러 가지 내장 메서드(기능)를 추가해 클래스로 정의해 놓은 것이다.
    * 주의할 점: String 객체는 내부 문자열을 변경할 수 없다. (내용이 바뀌는 것처럼 보이지만 사실은 메모리에 새로운 문자열 객체가 계속 생성되는 것임)

### 클래스와 객체 (Class & Object)
* 클래스: 객체를 만들기 위한 '설계도(틀)'이다. (예: 붕어빵 틀, TV 설계도)
* 객체: 설계도를 바탕으로 실제로 메모리에 만들어진 '제품'이다. (예: 붕어빵, 실제 TV)
* 자바에서는 new 키워드를 사용해 객체(인스턴스)를 생성한다. (Tv t = new Tv();)
* 객체의 구성 요소: 객체는 속성(변수)과 기능(메서드)으로 이루어져 있다.
   * 예: TV의 속성(채널, 전원 상태) + 기능(채널 바꾸기, 전원 켜고 끄기)

### 변수의 종류 (Variables)

변수는 선언된 위치와 static 키워드의 유무에 따라 3가지로 나뉜다. 이 차이를 아는 게 핵심이다.

* 인스턴스 변수 (iv): 클래스 영역에 선언되며, new를 통해 객체가 만들어질 때마다 생성된다. 객체마다 각각 다른 개별적인 상태를 저장할 때 쓴다.
* 클래스 변수 (cv): 인스턴스 변수 앞에 static이 붙은 변수다. 객체를 안 만들어도 자동으로 메모리에 올라가며, 모든 객체가 값을 공유하는 '공통 변수'다. (예: 카드 게임에서 모든 카드의 무대 넓이/높이)
* 지역 변수 (lv): 메서드 내부에서 선언되며, 메서드가 실행될 때 만들어졌다가 메서드가 끝나면 메모리에서 사라진다.

### 메서드와 JVM 메모리 구조 (Method & JVM)

메서드: 특정 작업을 수행하는 코드들을 하나로 묶어놓은 것이다. (반환 타입이 없으면 void를 쓴다.)

* 기본형 vs 참조형 매개변수
   * 기본형(Primitive): 메서드로 값을 넘겨줄 때 '값의 복사본'만 전달하므로, 메서드 안에서 값을 바꿔도 원래 원본 변수의 값은 변하지 않는다. (Read only)
   * 참조형(Reference): 메서드로 '값이 저장된 메모리 주소'를 넘겨주기 때문에, 메서드 안에서 값을 바꾸면 원본 변수의 값도 같이 바뀐다. (Read & Write)

### JVM의 메모리 구조

* Method Area: 클래스 데이터와 static 변수(클래스 변수)가 저장되는 공간.
* Call Stack (호출 스택): 메서드가 호출되면 실행에 필요한 메모리가 쌓이는 곳. 메서드가 끝나면 메모리가 자동으로 반환된다. (맨 위에 있는 메서드가 현재 실행 중인 메서드다.)
* Heap: new 연산자로 생성된 모든 객체(인스턴스)와 인스턴스 변수가 저장되는 거대한 공간.

### 오버로딩 (Overloading)

한 클래스 내에 이름이 똑같은 메서드를 여러 개 정의하는 것을 말한다. 대표적인 예로 자바의 System.out.println()이 있다. (무슨 값을 넣든 다 출력해 주니까 편리하다.)

* 오버로딩 성립 조건   
   * 메서드 이름이 무조건 같아야 한다.
   * 매개변수의 개수 또는 타입, 순서가 달라야 한다.

 주의할 점: 반환 타입(리턴 타입)은 오버로딩을 구분하는 데 아무런 영향을 주지 않는다

### 생성자 (Constructor)

인스턴스가 생성될 때 호출되는 '인스턴스 초기화 메서드'다. 객체를 만들 때 필수적인 데이터를 처음부터 채워 넣기 위해 쓴다.

특징: 클래스 이름과 완전히 똑같아야 하며, 리턴 타입이 아예 없다. (void도 안 적음)

기본 생성자 (Default Constructor): 클래스에 생성자가 하나도 정의되어 있지 않으면, 컴파일러가 아무 내용도 없는 기본 생성자(클래스명() {})를 자동으로 추가해 준다. 단, 생성자가 하나라도 내가 직접 만들었다면 자동 추가되지 않으니 주의해야 한다.

 * this와 this()의 차이
     * this: 인스턴스 자신을 가리키는 참조변수. 인스턴스 변수와 매개변수의 이름이 같아서 구별해야 할 때 주로 쓴다.
     * this(): 한 생성자에서 다른 생성자를 호출할 때 사용한다. 반드시 생성자의 첫 줄에서만 사용할 수 있다.




-------------------------------------------------

### 상속 (Inheritance)

기존의 클래스를 재사용해서 새로운 클래스를 만드는 개념이다. 자바에서는 extends 키워드를 사용한다.

부모 클래스가 상위(조상), 자식 클래스가 하위(자손)가 된다.

자식 클래스는 부모의 모든 멤버(필드, 메서드)를 물려받기 때문에, 부모보다 항상 멤버 개수가 같거나 많다. (생성자와 초기화 블록은 상속 안 됨!)

중요한 점: 자식 클래스들의 공통적인 부분은 부모 클래스에서 관리하고, 자식들은 자신만의 고유한 특징만 구현하면 되니까 코드 중복이 획기적으로 줄어들고 관리가 편해진다.

 * 상속 vs 포함
   * 상속 관계 (~은 ~이다): SportsCar is a Car. (스포츠카는 자동차이다. ➔ 상속)
   * 포함 관계 (~은 ~을 가지고 있다): Circle has a Point. (원은 점을 가지고 있다. ➔ 다른 클래스를 멤버 변수로 선언해서 사용)

### 오버라이딩 (Overriding)

부모 클래스로부터 상속받은 메서드의 내용을 자식 클래스 자신에게 맞게 새로 바꾸는 것(재정의)을 말한다.

* 오버라이딩 성립 조건
    * 메서드 이름이 같아야 한다.
    * 매개변수(타입, 개수)가 같아야 한다.
    * 반환 타입이 같아야 한다.

* 주의할 제약 사항
    * 접근 제어자는 부모 클래스의 메서드보다 좁은 범위로 변경할 수 없다. (부모가 protected면 자식은 protected나 public이어야 함)
    * 부모 클래스의 메서드보다 더 많은 예외(throws)를 선언할 수 없다.

### super와 super()

* super: 부모의 멤버와 내 멤버의 이름이 같을 때, 부모 것을 구별해서 부르기 위해 쓰는 참조변수.
* super(): 부모 클래스의 생성자를 호출할 때 쓴다. 자식 클래스의 생성자 첫 줄에는 반드시 this()나 super()를 호출해야 하는데, 안 적으면 컴파일러가 자동으로 super();를 첫 줄에 집어넣는다. (그래서 부모 클래스에 기본 생성자가 기본적으로 있어야 에러가 안 난다.)

### 제어자 (Modifiers)

클래스, 변수, 메서드의 선언부에 붙어서 특별한 의미를 부여하는 키워드들이다.

1. static, final, abstract

static (클래스의, 공통적인): 인스턴스를 안 만들어도 메모리에 먼저 올라간다. 멤버변수에 붙으면 모든 인스턴스가 공유하는 공통 변수가 된다.

final (마지막의, 변경할 수 없는): 변수에 붙으면 '상수(값 변경 불가)', 메서드에 붙으면 '오버라이딩 불가', 클래스에 붙으면 '상속 불가(자식을 가질 수 없음)'가 된다.

abstract (추상적인): 메서드에 붙으면 선언부만 있고 구현부 {}가 없는 추상 메서드가 되며, 클래스에 붙으면 추상 클래스가 된다. (인스턴스 생성 불가)

2. 접근 제어자 (Capsulation / 정보 은닉)
 
 데이터를 외부에서 함부로 바꾸지 못하게 막고, 복잡한 내부 구조를 감추기 위해 사용한다.

* public: 접근 제한 없음 (어디서나 가능)

* protected: 같은 패키지 내에서, 그리고 다른 패키지라도 자식 클래스라면 접근 가능

* (default): 아무것도 안 적은 상태. 같은 패키지 내에서만 접근 가능

* private: 오직 같은 클래스 내에서만 접근 가능 (외부 접근 원천 차단)

### 다형성 (Polymorphism)

객체지향에서 가장 중요한 개념 중 하나로, "조상 클래스 타입의 참조변수로 자손 클래스의 인스턴스를 참조할 수 있게 한 것"이다.


CaptionTv c = new CaptionTv(); // 일반적인 방법 (타입 일치)

Tv t = new CaptionTv();        // 다형성 활용 (조상 타입으로 자손 참조)


* Tv t = new CaptionTv();는 가능하지만, 역으로 자손 타입 변수로 조상 인스턴스를 참조하는 CaptionTv c = new Tv();는 절대 불가능하다. (자손의 기능 개수가 더 많아서 다룰 수 있는 범위를 넘어가기 때문)
* 참조변수의 타입에 따라 호출할 수 있는 멤버의 개수가 결정된다.

### 형변환 (Casting)

서로 상속 관계에 있는 클래스 사이에서만 형변환이 가능하다.

1. Up-casting (자손 ➔ 조상): 형변환 생략 가능 (Tv t = (Tv)c; 에서 조상 타입인 (Tv)는 생략 가능)
2. Down-casting (조상 ➔ 자손): 형변환 생략 불가능, 반드시 명시해야 함.

### instanceof 연산자

참조변수가 실제로 어떤 인스턴스를 가리키고 있는지 확인하기 위해 사용한다. 주로 Down-casting을 하기 전에 안전하게 타입을 체크하는 용도로 쓴다. (결과값은 true / false)

### 다형성의 실제 활용 예시

다형성을 배우는 진짜 이유는 코드의 확장성과 유연성 때문이다.

1. 매개변수의 다형성

메서드의 매개변수를 조상 타입(Product)으로 만들어 두면, 자손 타입(Tv, Computer, Audio)의 객체들이 새로 추가되더라도 메서드를 새로 만들 필요 없이 하나의 메서드로 전부 받아낼 수 있다.

예: void buy(Product p) 하나로 사내의 모든 제품 구매 처리 가능.

2. 여러 타입의 객체를 하나의 배열로 다루기

조상 타입의 객체 배열을 생성하면, 그 배열 하나에 서로 다른 자손 클래스들의 인스턴스를 한 번에 담아서 관리할 수 있다.

예: Product[] p = new Product[3]; p[0] = new Tv(); p[1] = new Computer();

### 예외 처리 (Exception Handling)

프로그램이 실행 도중 갑자기 죽는 것을 막고, 정상적인 상태를 유지하도록 처리하는 기법이다.

* 에러(Error) vs 예외(Exception)

    * 에러: JVM 메모리 부족(OOM)처럼 프로그램 코드로 수습할 수 없는 심각한 오류.
    * 예외: 코드 수습이 가능한 비교적 가벼운 오류.

* Exception 클래스들 (Checked 예외): 사용자의 실수 등 외적 요인으로 발생. 컴파일러가 예외 처리를 필수적으로 검사하므로 try-catch나 throws가 없으면 컴파일 에러가 난다. (예: 파일이 없을 때 발생하는 IOException)

* RuntimeException 클래스들 (Unchecked 예외): 개발자의 코드 실수로 발생. 컴파일러가 강제하지 않으므로 개발자가 주의해서 코드를 짜야 한다. (예: NullPointerException)

* 핵심 문법 정리
     * try-catch-finally: try 블록에서 에러가 나면 catch가 잡아서 해결하고, finally 블록은 에러 발생 여부와 상관없이 무조건 마지막에 실행된다. (주로 파일이나 자원을 닫을 때 필수 사용)
     * throws: 내가 예외를 직접 처리하지 않고, 이 메서드를 호출한 녀석에게 예외를 떠넘기는(토스하는) 방식이다.

### java.lang 패키지와 유용한 클래스

* Object 클래스 (모든 클래스의 조상)
  * equals(Object obj): 기본적으로는 '주소값'을 비교하지만, 보통 String처럼 내부의 '실제 값'을 비교하도록 우리가 재정의(오버라이딩)해서 쓴다.
  * hashCode(): 객체의 주소값으로 만든 해시코드다. equals를 오버라이딩할 때는 이 hashCode도 반드시 같이 오버라이딩해야 객체가 정상 작동한다.
* String vs StringBuffer / StringBuilder

   * String: 불변(Immutable) 객체다. 값을 한 번 정하면 바꿀 수 없어서, 글자를 더할 때마다 메모리에 새로운 문자열 객체가 계속 생성되는 낭비가 생긴다.
   * StringBuffer / StringBuilder: 가변(Mutable) 객체다. 내부 저장 공간(Buffer)을 가지고 있어서 문자를 자유롭게 지우고 수정할 수 있어 메모리가 절약된다. (멀티쓰레드 환경 안전 여부에 따라 안전하면 Buffer, 속도가 우선이면 Builder를 쓴다.)

### 컬렉션 프레임워크 (Collection Framework)

다수의 데이터를 쉽고 효과적으로 처리할 수 있게 자바가 표준화해 놓은 내장 자료구조 인터페이스다. 크게 3가지 핵심 갈래가 있다.

* List (순서 O, 중복 O): 대기 번호표처럼 데이터의 순서가 유지되고 똑같은 값도 여러 번 저장할 수 있다.
* ArrayList: 배열 구조라 조회는 엄청 빠르지만, 중간에 데이터를 끼워 넣거나 삭제할 때는 느리다.
* LinkedList: 기차 칸처럼 앞뒤 노드가 연결된 구조라 삽입/삭제는 번개처럼 빠르다.
* Set (순서 X, 중복 X): 집합 개념이다. 순서가 없고 중복을 허용하지 않는다. (예: 로또 번호 뽑기)
* HashSet: 중복을 체크할 때 내부적으로 equals()와 hashCode()를 호출해 확인한다.
* Map (Key-Value 쌍, 순서 X): 영한사전처럼 '단어(Key)'와 '뜻(Value)'이 짝을 이루는 구조다.

Key는 중복될 수 없고, Value는 중복이 가능하다. 만약 이미 있는 Key에 새로운 값을 넣으면 기존 값이 덮어써진다. (예: HashMap)

### 입출력 (I/O - Input / Output)

컴퓨터 내부나 외부의 장치와 데이터를 주고받는 흐름을 의미한다. 자바는 데이터를 이동시키는 통로로 스트림(Stream)이라는 개념을 쓴다.

단방향성: 입력 통로(InputStream)와 출력 통로(OutputStream)가 따로 존재하므로, 동시에 양방향으로 보낼 수 없다.

큐(FIFO) 구조: 먼저 들어간 데이터가 먼저 나오는 구조다.

* 바이트 기반 vs 문자 기반 스트림
   * 바이트 기반 (~InputStream, ~OutputStream): 그림, 음악, 동영상, 텍스트 등 모든 데이터를 1바이트 단위로 주고받는다.
   * 문자 기반 (~Reader, ~Writer): 오직 '텍스트(문자)' 데이터만 다루기 위한 스트림이다. 영어와 달리 한글 같은 문자는 2바이트 크기이기 때문에, 바이트 기반으로 처리하면 글자가 깨질 수 있어 문자 기반 스트림을 쓴다.

* 보조 스트림 
     * 실제 데이터를 직접 읽고 쓰는 기능은 없지만, 주 스트림에 붙여서 속도를 획기적으로 향상시켜 주는 버퍼 기능을 제공한다. (예: BufferedReader, BufferedInputStream)
